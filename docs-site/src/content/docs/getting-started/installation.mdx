Diagrammer can be installed as a desktop application or run directly in your browser.

## Desktop Application (Recommended)

The desktop version provides the best performance and full feature access including native file system integration.

<Tabs>
  <TabItem label="Windows">
    1. Download the latest `.msi` installer from the [releases page](https://github.com/QR-Madness/diagrammer/releases)
    2. Run the installer and follow the prompts
    3. Launch Diagrammer from the Start menu
  </TabItem>
  <TabItem label="Linux">
    **AppImage (Universal)**
    ```bash
    chmod +x Diagrammer-*.AppImage
    ./Diagrammer-*.AppImage
    ```

    **Debian/Ubuntu (.deb)**
    ```bash
    sudo dpkg -i diagrammer_*.deb
    ```

    **Fedora/RHEL (.rpm)**
    ```bash
    sudo rpm -i diagrammer-*.rpm
    ```

  </TabItem>
  <TabItem label="macOS (Must Build from Source)">
    Though your welcome to build from source following the instructions below, we do not yet provide official macOS installers yet.
  </TabItem>
</Tabs>

## Building from Source

For developers who want to build from source:

### Prerequisites

- [Bun](https://bun.sh) (package manager and runtime)
- [Rust](https://rustup.rs) (for Tauri backend)
- [Taskfile](https://taskfile.dev/#/installation) (optional but highly recommended; for task automation)
- Platform-specific dependencies (see [Tauri prerequisites](https://v2.tauri.app/start/prerequisites/))

### Build Steps

```bash
# Clone the repository
git clone https://github.com/QR-Madness/diagrammer.git
cd diagrammer

# Install dependencies
bun install

# Run in development mode
bun run dev          # Web only
bun run tauri:dev    # Desktop app

# Build for production
bun run build        # Web build
bun run tauri:build  # Desktop installer
```
